# AI Murder Mystery Detective

## What it is (1 sentence)
An AI-driven web game where players interrogate suspects in natural language to solve a uniquely generated murder mystery before time runs out.

## Track
[ ] Internal Tool   [ ] Fintech Mini-App   [ ] Crypto/Web3   [x] Free-form

## Target user (1 sentence)
Casual gamers and fans of detective fiction who enjoy mystery novels, films, or board games like Clue.

## The user's job-to-be-done (1 sentence)
Piece together who committed a murder — and why — purely through smart questioning, observation, and reasoning, with no hand-holding.

## Must-have features (3–5, no more)
1. **Google Auth + Coin Economy** — acceptance criteria: user logs in with Google, receives 1 000 starter coins, cannot play without login, coin balance persists across sessions.
2. **Case Generation (Easy / Medium / Hard)** — acceptance criteria: selecting a difficulty spends the correct coin amount (50 / 100 / 200), generates a unique case file (victim, crime scene image, 3–6 suspects), and starts a countdown timer (15 / 25 / 45 min).
3. **Natural-Language Suspect Interrogation** — acceptance criteria: player can type any question to any suspect; each suspect responds in a consistent voice reflecting their private truth, secrets, and personality.
4. **Hint System** — acceptance criteria: player can purchase one hint per case; hint deducts coins and returns a subtle, non-spoiling nudge; hint cost reduces final reward.
5. **Accusation & Reveal** — acceptance criteria: player selects a suspect and submits verdict; game reveals whether correct, shows full narrative resolution, and awards or deducts coins accordingly.

## Nice-to-have (won't block demo)
- Text-to-speech playback of the case file audio
- Crime scene image generated by AI (e.g. DALL-E / Stable Diffusion)
- Leaderboard of fastest solves per difficulty

## Tech stack
- Backend: TypeScript + NestJS 10 (Node 22) — separate service in `backend/`
- DB: Postgres 16 via Sequelize ORM + Sequelize migrations
- Frontend: Next.js 15 + Tailwind CSS + shadcn/ui — in `frontend/`
- Auth: Google OAuth handled by Express backend; JWT issued to frontend
- AI: Gemini free tier (Google Generative AI SDK)
- Monorepo: pnpm workspaces (`frontend/`, `backend/`)
- Platform-specific feature: Countdown timer with auto-submit on expiry

## Architecture (5 lines max)
Frontend is a Next.js App Router SPA (Tailwind, dark noir theme) that communicates exclusively with the Express backend over REST.
NestJS backend owns all logic: Google OAuth callback, JWT auth, four Gemini-backed modules (case generation, interrogation, hint, verdict).
Each suspect's private truth is stored in Postgres per session and never returned to the frontend.
Gemini generates the full case JSON once at game start; each interrogation turn injects the full case context server-side.
Coin balance and game state persist in Postgres; Railway hosts the backend, Vercel hosts the frontend.

## Out of scope
- Real-money coin purchases (coins are earned by play only)
- Multiplayer or co-op investigation
- Mobile native app (web-responsive only)

## Risks I see
1. **AI response consistency** — suspect lies and truths must stay coherent across multiple turns; mitigated by storing the full case JSON server-side and injecting it into every Claude prompt.
2. **Cost blow-up** — unlimited interrogation questions could spike Anthropic API spend; mitigated by per-session token budget and streaming with a hard stop.
3. **Image generation latency** — crime scene image may be slow on case load; mitigated by showing a placeholder and loading the image async, or using pre-generated assets for the demo.

## How I'll demo it (3 lines)
1. Judge logs in with Google, sees 1 000 coins, selects Medium difficulty (costs 100 coins), and watches the case file appear with victim details and suspects.
2. Judge interrogates two suspects via natural language, notices a contradiction, buys one hint, and uses it to confirm their suspicion.
3. Judge submits the accusation, sees the full narrative reveal with the true murderer and motive, and watches their coin balance update.
